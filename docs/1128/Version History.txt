Version History
---------------

06/04/2024      v3.5.0
                The Rfid.AsciiProtocol library has been updated to version 3.5.0. It is built with target SDK 34.
                This library implements the TSL ASCII Protocol v3.0.

                When a BT connection is lost, AsciiCommander will issue a ConnectionState.LOST event before the ConnectionState.DISCONNECTED event.

				The following are only supported when executed on an ASCII Protocol v3 Reader (e.g. 3166):

					All RF Commands e.g. Inventory, Read, Write etc.. can now specify halt conditions for the operation: the number of tags, the number of rounds and/or the duration.

					BarcodeCommand responses now include a BarcodeSymbology instance describing the scanned barcode.
				
					The ReadTransponderCommand now has a SelectPause parameter to provide the necessary (3ms) pause needed when reading Magnus S3 temperature Sensor Tags.
					
					The AlertCommand now has an AlertVolume parameter that allows the volume to be set from 0 to 100%.

					SystemCommand - see release v3.2.0
					LinkProfileCommand - see release v3.2.0

                Support added for the 3166 Reader.


10/08/2023      v3.3.0
                The Rfid.AsciiProtocol library has been updated to version 3.3.0. It is built with target SDK 33.
                This library implements the TSL ASCII Protocol v2.5.

                ConnectionStatus now extends com.uk.tsl.utils.Observable<ConnectionState> to avoid API Level 33 deprecation errors.
                This change requires that calls to ConnectionStatus.deleteObserver(...) are replaced with ConnectionStatus.removeObserver(...)

                Added method IAsciiTransport getTransport(TransportType type) to the Reader class. This is primarily for internal TSL test apps.


12/06/2023		v3.2.0
                The Rfid.AsciiProtocol library has been updated to version 3.2.0. It is built with target SDK 33.
                This library implements the TSL ASCII Protocol v2.5.
				
				Preliminary support for 3xxx series readers and ASCII Protocol v3.

				A SystemCommand has been added that can be used to alter the baud rate of the ePop-Loq connection. The default baud rate remains at 115200.
				An app can request a faster ePop-Loq rate to ensure that the additional throughput performance of the 3xxx series can be achieved.
				This is only supported on series 3xxx readers.

				The LinkProfileCommand has been added to allow the RF Mode, used to communicate with the tag, to be specified.
				This command is reader and region specific. It is not usually necessary to change this setting on Series 1xxx or Series 2xxx readers.
				For series 3xxx readers, the profile numbers are those listed in the STORM Protocol guide, under the $ir -rf parameter (RF Mode).


17/01/2023      v3.1.0
                The Rfid.AsciiProtocol library has been updated to version 3.1.0. It is built with target SDK 33.
                This library implements the TSL ASCII Protocol v2.5.

                This release fixes a bug that caused barcodes to be truncated when they contained line terminators 0x0D or 0x0A.

                Note: LocalBroadcastManager support is retained to allow users time to transition to the new approach.

                The Rfid.AsciiProtocol JavaDocs have been updated and now includes a search option.


21/10/2022      v3.0.0
                The Rfid.AsciiProtocol library has been updated to version 3.0.0 to meet the latest Google API requirements. It is built with target SDK 33.
                This library implements the TSL ASCII Protocol v2.5.

                To support the API31+ requirement of User permission checks, your app will need to issue the request for Bluetooth permissions where appropriate.
                If this is not done then Bluetooth readers will NOT BE AVAILABLE - e.g. calls to ReaderManager.sharedInstance().updateList() will return an empty list.

                https://developer.android.com/guide/topics/permissions/overview

                The new permissions to be requested are:

                    Manifest.permission.BLUETOOTH_CONNECT
                    Manifest.permission.BLUETOOTH_SCAN

                The library still provides the original permissions for older devices.

                At an appropriate point in your application, ask the user for the above permissions.
                When permissions are granted, call ReaderManager.sharedInstance().updateList() to ensure the reader list is up-to-date.

                The sample projects have all been updated with examples of how this can be done - there are slight differences in implementation details due to the differing UIs.

                The Sample code implementation asks for permission in the onStart() lifecycle method but you may find a better place in your own apps.

                    @Override
                    protected void onStart()
                    {
                        super.onStart();
                        checkForBluetoothPermission();
                    }

                All Sample Projects have been updated to build against API Level 33.

                To provide examples to developers starting new projects, the following have been re-written starting from the latest project templates
                (Android Studio Dolphin (Build #AI-213.7172.25.2113.9014738)):

                    Inventory
                    ReadWrite
                    ConfigureBarcode

                Note: In the Activity onCreate(), the creation & initialisation of AsciiCommander and ReaderManager classes now occurs before the layout is inflated.
                Inflation of the main UI triggers the lifecycle methods of the Fragment classes

                The above projects now make use of simple ViewBinding to access the UI elements.

                Bug Fixes:
                 - The API31+ requirement for specifying the mutability of PendingIntents was causing null pointer crashes with the previous library - these are now fixed.
